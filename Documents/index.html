APLICATIVO GESTION ADMINISTRATIVA PUNTOS EFECTY

Este proyecto de aplicativo web para la gestión administrativa de puntos efecty está basado en micro servicios, por tanto el desarrollo del aplicativo se realizara de modularmente independiente de si su integración con el resto de módulos.

Por la naturaleza del proyecto este estará sometido a cambios regularmente durante su elaboración, por tal motivo se realizara por medio de metodología de desarrollo ágil, la escogida es Extreming Programing. La Extreming Programing o XP, se basa en el desarrollo iterativo e incremental, se realizan tareas en tiempos cortos y al final cada uno se debe tener un producto terminado, de esta manera en muy poco tiempo se puede tener el aplicativo en funcionamiento e ir agregando los micro servicios faltantes, osa agregar y o modifica funcionalidades a medida que pase el tiempo.

La tareas a realizar se designan o surgen atreves de las historias de usuarios, por tanto el desarrollo que cada funcionalidad del aplicativo será dada por una historia de usuario, estas son analizadas para de ella obtener las iteraciones correspondientes para su realización, el resumen una historia de usuario debe ser dada por la enunciación de una prueba que falle.

De esta manera se logra y obteniendo el aplicativo funcional a medida que se va desarrollando y no esperar hasta el final del proyecto. Después de la refactorización la historia debe seguir pasando la prueba, este tipo de metodología de desarrollo se implemente para cada parte de las cosas que se hagan en el proyecto, no se puede dejar de lado el que una historia de usuario tendrá varias tareas a realizar por tanto cada tarea de debe realizar con el mismo tipo de metodología.

Tecnologia a Usar

Para el cumplimiento de las historias de usuario el primer enfoque sera el diseño , esto dado que una historia de usuario expresa el porque y que se quiere hacer, ahora el siguiente paso es el como, este esta dividido en dos partes. Como se hace y como hacerlo.

  • Como se hacer, es referente a que pasos deve seguir el cliente para lograr la funcionalidad que pidio
  • Como hacerlo, es la tecnologia que use para desarrollar lo cometido

Recordar que el Extreming Programing se basa mas es el Cliente, dado que una se sus valores es que el cliente y el desarrollador este siempre en contacto, para que exista una armonia es claro que un Cliente no pude tener una buena comunicacio con un desarrollador tecnologico , pero si se entendera mejor con el desarrollador de diseño o de experiencias de usuario, el resultado del trabajo de estos ultimos seguro sera mejor digerido por el cliente, que el de un "Programador" que le muestre el codigo de las dependencias a usar; En la elaboracion de proyectos se realiza el Manual Tecnico y El Manuel de Usario este es el mas usado.

Si el Extreming Programing deja de lado , el preocuparse por el conocer todas las funcionalidades que tendra la aplicacion, porque entonces no ocuparse primero en como probablemente se vera esa funcionaliadad y luego el como hacerlo posible, asi no se perdera el tiempo en hacer codigo sobre cosas que no existiran o seran usadas en esa funcion en especifico.

CAPITULO 1

Un cajero de un punto de atención Servientrega-Efecty, que por cuestiones de su labor como tal, debía llevar un registro del dinero que tenía en caja cada cierto tiempo y así llevar las cuentas claras , esa acción se llamaba Arqueo de Caja, por medio de esta él contaba todo el dinero que tenía en su caja y realizaba la diferencia con el que el sistema que este manejaba de mostraba, para que esa tarea fuera más fácil y además por pautas laborales el cada cierto tiempo armaba paquetes de dinero y los sacaba se su caja, por ende no tenía todo el dinero que había manejado en el trascurso de su labor en caja, solo una parte, y otra previamente contada en La Caja Fuerte , de esta manera al momento de realizar el Arqueo, SOLO tenía que sumar los paquete de dinero más lo que tenía en caja, el registro de esta acción era anotado para su posterior uso.

Historia 1 :

El cajero debe poder realizar suma del dinero sin diferencia por denominaciones de dinero.
